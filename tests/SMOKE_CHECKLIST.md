# 冒烟测试清单

> 每次部署云函数后，逐项验证以下核心流程。
> ⚠️ 标记为最高优先级的项目请务必验证。

## 部署前检查

- [ ] `npm run check:sync` — common 模块同步一致性检测通过
- [ ] `npm run test:all` — 全量单元测试通过

---

## 核心流程

### 1. 注册/登录
- [ ] 新手机号注册 → 门店码登录 → 验证进入首页
- [ ] 已注册手机号登录 → 直接进入首页
- [ ] 游客模式（13507720000/123456）→ 可浏览体验数据

### 2. 开单流程（核心）⚠️
- [ ] 选择车辆 → 添加服务项目 → 输入金额 → 选择关联商品 → 完成
- [ ] 工单在列表可见，工单详情数据正确
- [ ] 分享工单图片生成正常

### 3. 编辑工单
- [ ] 修改服务项目/金额 → 保存 → 详情页验证修改已生效

### 4. 作废工单
- [ ] 已完成工单作废 → 列表标记已作废 → 库存回滚正确

### 5. 新增车辆
- [ ] 输入车牌号 → 保存 → 车辆列表可见

### 6. 会员权益
- [ ] 添加会员 → 设置权益 → 查看权益详情
- [ ] 核销权益 → 权益剩余次数正确扣减

### 7. OCR 车牌识别 ⚠️ 最高优先级
- [ ] 拍照 → 识别结果正确 → 自动填入车牌输入框
- [ ] 无图片 → 提示"缺少图片数据"
- [ ] 无法识别的图片 → 提示"未识别到车牌"

### 8. OCR 车架号识别 ⚠️ 最高优先级
- [ ] 拍照 → 识别结果正确 → 自动填入VIN输入框
- [ ] 无图片 → 提示"缺少图片数据"

### 9. AI 月报生成 ⚠️ 最高优先级
- [ ] 进入月报页面 → 自动生成或查看已有报告
- [ ] 收入、工单数、客单价、新客占比等指标正确
- [ ] 健康评分雷达图正确显示

### 10. AI 月报诊断 ⚠️ 最高优先级
- [ ] 诊断项合理（如"新客获取不足"/"增值服务渗透不足"等）
- [ ] 案例图片可点击查看
- [ ] 无数据门店提示友好

### 11. 进销存
- [ ] 新增商品 → 入库 → 库存数量正确
- [ ] 开单关联商品 → 库存自动扣减
- [ ] 流水记录正确

### 12. 数据导出
- [ ] 车辆导出 → 数据完整
- [ ] 会员导出 → 数据完整
- [ ] 工单导出 → 数据完整
- [ ] 库存流水导出 → 数据完整

### 13. 员工管理
- [ ] 添加员工 → 列表可见 → 员工可登录
- [ ] 修改角色 → 权限变化正确
- [ ] 移除员工 → 员工无法再登录

### 14. 报表页面
- [ ] 今日/本周/本月/本年 Tab 切换正常
- [ ] 数据统计正确（收入/工单数/客户数/排名）
- [ ] 下拉刷新 → 数据强制重拉
