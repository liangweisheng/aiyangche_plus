### 2.6 发现的问题

#### BUG#1 🐛 — `repair_inventory/index.js` admin 权限判断逻辑错误（严重）

**位置**：L106-111
```javascript
if (requireAdmin) {
  var role = callerRecord ? (callerRecord.role || 'admin') : ''
  if (role !== 'admin' && !callerRecord) {    // ← 此处应为 || 而非 &&
    return { code: -5, msg: '无权限，仅管理员可操作' }
  }
}
```

**根因**：条件使用 `&&` 而非 `||`。当 `callerRecord` 存在且 `role === 'staff'` 时：
- `role !== 'admin'` → `true`
- `!callerRecord` → `false`
- `true && false` → `false` → **通过鉴权**

**影响**：`staff` 角色可以调用以下 admin-only action：
- `addProduct` / `updateProduct` / `addStock` / `adjustStock` / `batchAddStock` / `getStockLogs` / `savePhrases` / `toggleProductStatus` / `importTemplateProduct` / `batchImportTemplates` / `updateStockLogOrderId`

**修复方案**：将 `&&` 改为 `||`

---

#### BUG#2 ✅ — `batchGenerateMonthlyReports` 数据库查询 `.and()` 方法不存在（已修复）

**原位置**：`repair_main/index.js` L1839（已迁至 `repair_aux/index.js`）

**原 Bug**：
```javascript
createTime: _.gte(monthStart).and(_.lte(monthEnd))  // ❌ .and() 不存在
```

**修复后**（repair_aux 中 3 处全部已修复）：
- L519 `generateMonthlyReport`：`_.and([_.gte(monthStart), _.lte(monthEnd)])`
- L606 `generateMonthlyReport`（上月对比）：`_.and([_.gte(prevMonthStart), _.lte(prevMonthEnd)])`
- L818 `getMonthlyReport`（前置检查）：`_.and([_.gte(_rmStart), _.lte(_rmEnd)])`
- L1057/L1073 `batchGenerateMonthlyReports`：统计阶段同样使用正确语法

**验证**：`repair_aux` 目录中无任何 `.gte(x).and(` 模式，全部使用 `_.and([...])` ✅

---

#### 建议#1 — `repair_inventory` 库存扣减操作建议添加事务保护

`_updateSpecStock` 使用读-改-写模式（先 `get` 再 `update`），高并发场景下两个请求同时读取到同一库存值会导致超卖。建议：
- 改用 `_.inc(-delta)` 原子操作（但需要先将 specStock 转为对象结构支持原子更新）
- 或添加乐观锁（版本号校验）
