### 3.3 免费版限制规则

#### 工单数量限制

- **规则**：免费版累计最多开 100 个工单（非日限）
- **计数范围**：从注册到激活 Pro 之前的所有历史工单总计，永不重置
- **触发点**：
  - `orderAdd` 开单保存前：查询 `repair_orders` 累计 count，≥100 时弹窗拦截禁止保存，引导去 Pro 激活
  - `dashboard` 看板：累计 ≥100 时显示"已达上限，请升级 Pro"提示
- **解除方式**：激活 Pro 版（`code` 有值 + `expireTime` 未过期）后所有限制解除

#### 报表页限制

- **免费+admin**：仅能查看"今日" tab，本周/本月/本年三个 tab 不加载数据，点击时弹窗提示升级
- **staff**：直接隐藏报表 tab（看不到入口）

### 3.4 Pro 状态继承机制

员工的 Pro 状态**不存储在员工记录上**，而是**继承自店主**。

```
员工登录
  → 获取员工记录 → 读取 shopPhone（店主手机号）
  → 用 shopPhone 查询 repair_activationCodes where phone=店主手机 AND role='admin'
  → 读取店主记录的 code + expireTime
  → 用 _checkProStatus(店主记录) 判断 Pro 状态
  → 缓存 isPro 到本地
```

**约束**：只有 Pro 版门店才能使用"员工管理"功能（UI 隐藏 + 云函数拦截双重保障）。

### 3.5 员工管理流程

#### 添加员工

```
管理员 → 我的页面 → 员工管理 → 邀请员工
  → 输入员工手机号
  → 选择角色：○ 店员(staff)  ● 管理员(admin)
  → 点击"发送邀请"
  → 云函数 addStaff 校验：
    1. 操作者是否为 Pro 版（非 Pro 拒绝）
    2. 员工手机号格式是否合法
    3. 员工是否已存在
  → 创建 repair_activationCodes 记录（role/staffOpenid/addedBy/addedTime/status）
```

#### 员工登录

```
员工打开小程序 → 登录页
  → 输入手机号 + 门店码
  → 查询 repair_activationCodes where phone=员工手机
  → 找到记录 → 读取 role
  → 缓存 role + shopPhone 到本地
  → 通过 shopPhone 继承店主 Pro 状态
  → 自定义 TabBar 根据 role 显示对应 tab
```

### 3.6 数据库变更

#### repair_activationCodes 集合新增字段

| 字段 | 类型 | 说明 | 适用角色 |
|------|------|------|---------|
| role | String | `'admin'` 或 `'staff'` | 全部记录 |
| staffOpenid | String | 员工的 openid | staff 记录 |
| addedBy | String | 添加者的手机号（管理员） | staff 记录 |
| addedTime | Date | 添加时间 | staff 记录 |
| status | String | `'active'` 或 `'disabled'` | staff 记录 |

> 注册者记录已有字段全部保留不变，仅新增 `role: 'admin'`。

#### 数据存储示例

```javascript
// 店主记录（注册时自动生成）
{
  _id: "abc123",
  name: "张氏汽修",
  phone: "139xxxx5678",        // 店主手机 = shopPhone（数据隔离主键）
  openid: "oXXXX_店主微信",
  shopCode: "A3K9M2",
  role: "admin",               // 注册时自动设置
  code: "PRO-2026-XXXX",       // Pro 激活码（激活后写入）
  expireTime: Date,            // Pro 到期时间
  type: "free",
  createTime: Date
}

// 员工记录（管理员添加时生成）
{
  _id: "def456",
  phone: "138xxxx1234",        // 员工自己的手机
  openid: "oXXXX_员工微信",
  shopCode: "A3K9M2",          // 与店主相同
  shopPhone: "139xxxx5678",    // 指向店主手机（数据隔离+Pro继承）
  role: "staff",               // 管理员指定
  addedBy: "139xxxx5678",      // 谁添加的
  addedTime: Date,             // 添加时间
  status: "active"             // 启用/停用
}
```

> **关键设计**：`shopPhone` 是数据隔离的核心——所有业务查询都用 `shopPhone` 过滤，同门店的所有人看到同一份数据。

### 3.7 云函数变更

#### registerShop action 修改（仅加1行）

在新增记录时自动设置 `role: 'admin'`：


#### 新增 action（纯增量）

| action | 功能 | 关键参数 | 说明 |
|--------|------|---------|------|
| addStaff | 添加员工 | staffPhone, role | 校验 Pro 状态、手机号格式、是否已存在 |
| removeStaff | 移除员工 | staffPhone | 将员工记录 status 改为 'disabled' |
| updateStaffRole | 修改员工角色 | staffPhone, newRole | 仅 admin/staff 两种值 |
| listStaffs | 查询员工列表 | 无 | 返回当前门店所有 staff 记录 |

**addStaff 核心逻辑**：

```javascript
async function addStaff(event, openid) {
  var staffPhone = event.staffPhone || ''
  var staffRole = event.role || 'staff'

  // 1. 校验操作者 Pro 状态
  var adminRecord = await db.collection('repair_activationCodes')
    .where({ openid: openid, type: 'free' }).limit(1).get()
  var adminData = adminRecord.data[0]
  // _checkProStatus 判断 code+expireTime
  if (!adminData.code || _isExpired(adminData.expireTime)) {
    return { code: -4, msg: '请先升级 Pro 版本才能添加员工' }
  }

  // 2. 创建员工记录
  await db.collection('repair_activationCodes').add({
    data: {
      phone: staffPhone,
      shopCode: adminData.shopCode,      // 继承门店码
      shopPhone: adminData.phone,        // 指向店主
      role: staffRole,
      addedBy: adminData.phone,
      addedTime: db.serverDate(),
      status: 'active',
      type: 'free',
      createTime: db.serverDate()
    }
  })

  return { code: 0, msg: '添加成功' }
}
```

## 七、风险点

| 风险 | 严重程度 | 应对策略 |
|------|---------|---------|
| 自定义 TabBar 兼容性 | 中 | 使用微信官方 custom-tab-bar 规范，保留回滚路径 |
| CloudBase Web SDK 浏览器兼容性 | 中 | 先做 POC 验证，确认 SDK 能正常连上云环境 |
| file:// 协议安全限制 | 低 | 提供 start.bat 脚本启动本地 HTTP 服务 |
| 数据库安全规则（PC端访问） | 中 | 需在云开发控制台配置允许匿名登录访问指定集合 |
| 小程序和 PC 同时操作同一数据 | 低 | 关键操作由云函数保证事务性 |
