# 微信小程序跨账号云环境共享 - 开发指南

> 适用场景：多个小程序共用同一个云开发环境（数据库、云函数、云存储等），实现数据统一管理和资源复用。

---

## 一、架构概述

```
┌─────────────────────┐        ┌─────────────────────────────────┐
│   调用方小程序 A    │        │         资源方小程序 B           │
│  （使用方 / 本项目）  │        │    （拥有云环境的一方）           │
│                     │        │                                 │
│  app.js 初始化      │───────▶│  cloudbase_auth 云函数（鉴权）    │
│  wx.cloud.Cloud()  │        │  cloud1-xxxxxxxxxxxxxx          │
│  ↓                  │        │                                 │
│  数据库 / 云函数     │◀──────│  repair_main（聚合云函数）        │
│  读写操作           │        │  repair_cars / orders / ...     │
└─────────────────────┘        └─────────────────────────────────┘
```

- **调用方**：当前小程序，通过 `wx.cloud.Cloud` 跨账号访问资源方的云环境
- **资源方**：实际拥有云开发环境的小程序，需部署 `cloudbase_auth` 鉴权函数

---

## 二、前置准备

### 2.1 资源方配置（一次性操作）

1. **资源方小程序**开通云开发，获得环境 ID（如 `cloud1-xxxxxxxxxxxxxx`）
2. **资源方云开发控制台** → 设置 → 环境共享 → 添加调用方小程序的 AppID
3. **资源方部署 `cloudbase_auth` 云函数**（见下方代码）

### 2.2 调用方配置

1. **调用方小程序**也需开通云开发（但可以不使用自己的环境，仅用于 `wx.cloud.init`）
2. 在 `project.config.json` 中配置云函数根目录（`cloudfunctionRoot`）

### 2.3 所需参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `resourceAppid` | 资源方小程序的 AppID | `wxb1c736174ede330c` |
| `resourceEnv` | 资源方云环境 ID | `cloud1-2gwoxtay6a4d8181` |

## 三、核心实现

### 3.1 app.js - 跨账号初始化

```javascript
// app.js 顶部：模块级变量
var RESOURCE_APPID = 'wxb1c736174ede330c'  // 资源方 AppID
var RESOURCE_ENV = 'cloud1-2gwoxtay6a4d8181' // 资源方云环境 ID

var _resourceDb = null       // 缓存数据库实例
var _cloudReady = false      // 云端是否就绪
var _cloudInitPromise = null // 初始化 Promise（用于异步等待）

App({
  onLaunch() {
    // 1. 本地云环境初始化（必须，否则 wx.cloud.Cloud 无法使用）
    wx.cloud.init({ traceUser: true })

    // 2. 初始化跨账号资源共享
    this._initResourceCloud()
  },

  /**
   * 初始化跨账号资源共享
   * 通过 cloudbase_auth 云函数进行鉴权
   */
  _initResourceCloud() {
    var app = this
    _cloudInitPromise = new Promise(function (resolve, reject) {
      app._resourceCloud = new wx.cloud.Cloud({
        resourceAppid: RESOURCE_APPID,
        resourceEnv: RESOURCE_ENV
      })
      app._resourceCloud.init().then(function () {
        _resourceDb = app._resourceCloud.database()
        _cloudReady = true
        console.log('跨账号资源共享初始化成功')
        resolve()
      }).catch(function (err) {
        console.error('跨账号资源共享初始化失败', err)
        reject(err)
      })
    })
  },

  /**
   * 获取跨账号数据库实例
   */
  db: function () {
    return _resourceDb
  },

  /**
   * 等待云端就绪（解决异步时序问题）
   * tabBar 页面在 onShow 中调用后，再进行数据加载
   */
  whenCloudReady: function () {
    if (_cloudReady) return Promise.resolve()
    return new Promise(function (resolve) {
      var timer = setInterval(function () {
        if (_cloudReady) {
          clearInterval(timer)
          resolve()
        }
      }, 100)
      // 超时保护 10 秒
      setTimeout(function () {
        clearInterval(timer)
        resolve()
      }, 10000)
    })
  }
})
```

### 3.2 cloudbase_auth 云函数（部署在资源方）

> 此函数必须部署在**资源方**的云环境中，不是调用方的。

目录结构：
```
cloudfunctions/
  cloudbase_auth/
    index.js
    package.json
```

**index.js：**
```javascript
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  return {
    errCode: 0,
    errMsg: '',
    auth: JSON.stringify({ access: 'granted' })
  }
}
```

**package.json：**
```json
{
  "name": "cloudbase_auth",
  "version": "1.0.0",
  "description": "跨账号资源共享鉴权",
  "main": "index.js",
  "dependencies": {
    "wx-server-sdk": "~2.6.3"
  }
}
```

**部署方式**：在资源方小程序项目中，右键 `cloudbase_auth` 文件夹 → 上传并部署（云端安装依赖）

## 四、前端调用方式

### 4.1 数据库操作

所有页面的数据库操作统一使用 `app.db()` 获取跨账号实例：

```javascript
// ✅ 正确：在函数内部调用 app.db()
Page({
  onLoad() {
    var app = getApp()
    app.whenCloudReady().then(function() {
      loadData()
    })
  },

  loadData() {
    var app = getApp()
    var db = app.db()
    var shopPhone = app.getShopPhone()

    db.collection('repair_orders')
      .where({ shopPhone: shopPhone })
      .orderBy('createTime', 'desc')
      .limit(20)
      .get({
        success: function(res) {
          console.log('查询结果:', res.data)
        }
      })
  }
})
```

```javascript
// ❌ 错误：模块顶层调用（App 未初始化完成，db 为 null）
var db = app.db()  // 真机会报错：Cannot read properties of null
```

### 4.2 云函数调用

云函数通过跨账号实例调用：

```javascript
var app = getApp()
var cloud = app._resourceCloud || wx.cloud  // 优先用跨账号实例

cloud.callFunction({
  name: 'repair_main',
  data: {
    action: 'createOrder',
    plate: '粤B12345',
    shopPhone: app.getShopPhone()
    // ... 其他业务参数
  },
  success: function(res) {
    if (res.result && res.result.code === 0) {
      console.log('操作成功:', res.result.data)
    } else {
      console.warn('业务错误:', res.result.msg)
    }
  },
  fail: function(err) {
    console.error('云函数调用失败:', err)
  }
})
```

### 4.3 tabBar 页面异步等待

tabBar 页面可能在跨账号初始化完成前就已加载，必须使用 `whenCloudReady()`：

```javascript
Page({
  onShow() {
    var app = getApp()
    app.whenCloudReady().then(function() {
      // 云端就绪后再加载数据
      app.getOpenId().then(function(openid) {
        if (openid) {
          loadData()
        }
      })
    })
  }
})
```

## 五、聚合云函数（路由模式）

将多个业务函数合并为一个云函数，通过 `event.action` 路由分发，减少云函数数量。

### 5.1 路由入口

```javascript
// cloudfunctions/repair_main/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

const db = cloud.database()
const _ = db.command

// 引入各业务模块（可直接写在同文件或 require 外部文件）
// async function registerShop(event, openid) { ... }
// async function createOrder(event, openid) { ... }

exports.main = async (event, context) => {
  var action = event.action || ''
  if (!action) return { code: -1, msg: '缺少 action 参数' }

  var wxContext = cloud.getWXContext()
  var openid = wxContext.OPENID

  var handler = {
    registerShop: registerShop,
    createOrder: createOrder,
    addCar: addCar,
    addMember: addMember,
    getDashboardStats: getDashboardStats,
    // ... 更多 action
  }

  if (!handler[action]) {
    return { code: -1, msg: '未知的 action: ' + action }
  }

  return handler[action](event, openid)
}
```

### 5.2 业务函数模板

```javascript
async function createOrder(event, openid) {
  // 1. 从 event 取参数
  var plate = event.plate || ''
  var shopPhone = event.shopPhone || ''

  // 2. 参数校验
  if (!plate) return { code: -1, msg: '车牌号不能为空' }

  // 3. 业务逻辑
  try {
    var addResult = await db.collection('repair_orders').add({
      data: {
        plate: plate,
        shopPhone: shopPhone,
        _openid: openid,
        createTime: db.serverDate()
      }
    })

    // 4. 返回统一格式
    return {
      code: 0,         // 0 = 成功
      msg: '操作成功',
      data: { _id: addResult._id }
    }
  } catch (err) {
    console.error('createOrder 错误:', err)
    return { code: -99, msg: '操作失败，请重试' }
  }
}
```

### 5.3 统一返回格式

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | number | `0` 成功，`-1` 参数错误，`-99` 系统异常 |
| `msg` | string | 提示信息 |
| `data` | object | 返回数据（成功时） |

## 六、注意事项与踩坑记录

### 6.1 异步时序问题

**现象**：tabBar 页面报 `Cannot read properties of null (reading 'command')`

**原因**：`_initResourceCloud()` 是异步操作，tabBar 页面的 `onLoad/onShow` 可能在初始化完成前执行

**解决**：所有 tabBar 页面使用 `whenCloudReady()` 等待初始化完成

### 6.2 模块顶层调用问题

**现象**：真机报 `env not exists` 或 `Cannot read properties of null`

**原因**：在 JS 文件顶层写 `const db = app.db()`，此时 App 尚未初始化

**解决**：将 `app.db()` 放在函数内部调用，不要在模块顶层获取

### 6.3 Promise 中 this 指向问题

**现象**：`Cannot read property '_resourceCloud' of undefined`

**原因**：Promise 回调中的 `this` 不是 App 实例

**解决**：在 Promise 外部用 `var app = this` 缓存引用

### 6.4 云函数部署环境

- `cloudbase_auth` 必须部署在**资源方**云环境
- `repair_main` 等业务云函数也部署在**资源方**云环境
- 调用方通过 `app._resourceCloud.callFunction()` 跨账号调用
- 部署时需使用**资源方 AppID** 登录的微信开发者工具

### 6.5 版本过渡期

新旧版本共存时的注意事项：

1. 旧版云函数不要过早删除，等新版本覆盖率达到要求后再清理
2. 审核期间用户操作的数据会分散在两个环境
3. 保留旧环境数据备份，以备回退需要

## 七、快速接入 Checklist

新项目接入跨账号云环境共享，按以下步骤操作：

- [ ] 获取资源方的 `AppID` 和云环境 `Env ID`
- [ ] 资源方控制台添加当前小程序 AppID 到环境共享白名单
- [ ] 资源方部署 `cloudbase_auth` 云函数
- [ ] 当前项目 `app.js` 添加 `_initResourceCloud()` / `db()` / `whenCloudReady()` 方法
- [ ] 所有页面将 `wx.cloud.database()` 替换为 `app.db()`
- [ ] 云函数调用改为 `(app._resourceCloud || wx.cloud).callFunction()`
- [ ] tabBar 页面数据加载包裹 `whenCloudReady()`
- [ ] 检查模块顶层无 `app.db()` 调用
- [ ] 真机调试验证数据库读写和云函数调用正常
