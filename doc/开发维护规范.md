
### WXML 中的硬编码值

WXML 无法直接引用 JS 常量，需通过 `setData` 传递：

```javascript
// 在 onLoad 或合适生命周期中设置
this.setData({
  freeMaxOrders: constants.FREE_MAX_ORDERS,
  freeMaxMembers: constants.FREE_MAX_MEMBERS,
  servicePhone: constants.SERVICE_PHONE
})
```

```html
<!-- WXML 中使用 -->
<text>已使用 {{usedCount}} / {{freeMaxOrders}} 单</text>
```

## 3. 页面生命周期规范

### 3.1 `onShow` 首次双重请求问题

微信小程序生命周期中 `onShow` 总是在 `onLoad` 后触发。若不做守卫，首次进入页面会发起**双重网络请求**。

**标准模式：**

```javascript
onLoad(options) {
  this._firstLoad = true
  this.fetchData()
},

onShow() {
  if (this._firstLoad) {
    this._firstLoad = false
    return      // ← 跳过首次，onLoad 已拉取
  }
  this.fetchData()  // ← 仅从子页面返回时刷新
}
```

### 3.2 `onPullDownRefresh` 异步等待

下拉刷新后应**等待数据加载完成**再调用 `wx.stopPullDownRefresh()`：

```javascript
// ✅ 正确
onPullDownRefresh() {
  this._resetAndFetch()
  // fetch 内部成功后或超时后停止动画
}

// ❌ 错误：不等异步完成就停止
onPullDownRefresh() {
  this.fetchList(true)
  wx.stopPullDownRefresh()  // 立即执行，用户看不到刷新动画
}
```

## 4. 门店数据隔离

### 4.1 核心原则

所有数据库查询**必须**通过 `app.shopWhere()` 添加 `shopPhone` 条件，确保数据仅限本门店可见。

### 4.2 开发规范

```javascript
// 读取单条记录：必须校验归属
fetchDetail(id) {
  db.collection('collection').doc(id).get({
    success: (res) => {
      const shopPhone = app.getShopPhone()
      if (res.data.shopPhone && res.data.shopPhone !== shopPhone) {
        wx.showToast({ title: '无权查看', icon: 'none' })
        setTimeout(() => wx.navigateBack(), 1500)
        return
      }
      // ... 正常渲染
    }
  })
}

// 列表查询：始终包裹 shopWhere
fetchList() {
  const where = app.shopWhere({})  // 空对象先拿基础条件
  if (keyword) where.plate = db.RegExp({ regexp: keyword, options: 'i' })
  db.collection('collection').where(where).get()
}
```

## 5. 异步流控制

### 5.1 Promise 链完整性

所有异步操作必须通过 Promise 串联：

```javascript
// ✅ 正确：返回值经 Promise 链传递
_fetchAndExportCars() {
  return page._fetchAll('repair_cars', where).then(function (cars) {
    return page._generateAndShare(rows, fileName)
  })
}

// ❌ 错误：不返回 Promise，调用方提前结束
_generateAndShare(rows) {
  fs.writeFile({ success: () => page._doShare() })  // 没有 return
}
```

### 5.2 `setData` 后的数据读取

微信小程序 `setData({ 'a.b': newVal })` 会**创建新的 `a` 对象**。如果在 `setData` 前用变量捕获了 `this.data.a`，该变量**指向旧对象，读不到新值**。

```javascript
// ❌ 错误
var data = this.data          // 捕获整个 data
this.setData({ 'form.total': '100' })  // this.data.form 现在是新对象
var val = data.form.total     // ← 读的是旧对象，值为空!

// ✅ 正确：setData 后重新读取
this.setData({ 'form.total': '100' })
var val = this.data.form.total  // ← 读最新值
```

## 6. 权限控制一致性

### 6.1 权限层级

| 角色 | 权限范围 |
|------|---------|
| 超级管理员 (`isOwner`) | 全部功能：编辑门店信息、数据导出、员工管理、AI诊断设置 |
| 管理员 (`role: admin`) | 除超级管理员专属外的全部功能 |
| 店员 (`role: staff`) | 仅核销/开单/新增车辆，不可访问报表和"我的"页面 |
| 游客 | Dashboard 只读演示 |

### 6.2 开发规范

所有写操作入口（编辑、删除、保存）必须检查角色权限：

```javascript
onEditXxx() {
  if (getApp().isStaff()) {
    wx.showToast({ title: '店员无权编辑', icon: 'none' })
    return
  }
  // ... 正常编辑逻辑
}
```

## 7. WXML 数据绑定规范

### 7.1 JavaScript Falsy 值陷阱

WXML 中的 `||` 运算符与 JS 一致：`0`、`''`、`false`、`null`、`undefined` 均为 falsy。

```html
<!-- ❌ 错误：score 为 0 时显示 "--" -->
<text>{{report.healthScore.total || '--'}}</text>

<!-- ✅ 正确：在 JS 中预处理 -->
<!-- JS: score._displayTotal = score.total != null ? String(score.total) : '--' -->
<text>{{report.healthScore._displayTotal}}</text>
```

### 7.2 深层属性访问的崩溃风险

```html
<!-- ❌ 错误：checkItems 中缺失 key 时，undefined.normal 崩溃 -->
<view class="{{detail.checkItems[item.key].normal ? 'normal' : 'warn'}}">

<!-- ✅ 正确：先判断父级存在 -->
<view class="{{detail.checkItems[item.key] && detail.checkItems[item.key].normal ? 'normal' : 'warn'}}">
```

### 7.3 数组 null 防护

在 JS 中处理返回数据时：

```javascript
// ❌ 错误
var items = res.data.map(function (item) { ... })

// ✅ 正确
var items = (res.data || []).map(function (item) { ... })
```

## 9. 云函数优化方向

### 9.1 当前架构

```
repair_main (路由模式)
├── registerShop / activatePro / addCar / addMember
├── createOrder / editOrder / voidOrder / useBenefit
├── updateShopInfo / updateMember / updateCarInfo / updateOpenid
├── getDashboardStats / getTotalSpent / getCarOrderStats
├── generateMonthlyReport / getMonthlyReport / listRecentReports
├── getShopProfile / updateShopProfile
├── addStaff / removeStaff / updateStaffRole / listStaffs
├── saveCheckSheet / getOpenId
└── loginByPhoneCode (多端)
```

### 9.2 可优化点

| 优化项 | 说明 |
|--------|------|
| **冷启动时间** | 单个云函数包含 20+ action，首次调用需加载全部逻辑，可拆分为 3-4 个独立云函数 |
| **index.js 体积** | ~2000 行，按功能模块拆分后更易维护 |
| **日志方案** | 建议统一使用 `console.log` 前缀标识（如 `[repair_main] addCar:`），便于日志检索 |

> ⚠️ **v4.0.0 红线约束：** 现有 action 的入参/出参不可变更，新增必须是纯增量
