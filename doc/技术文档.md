### 1.2 跨账号资源共享架构

运营方小程序通过 `new wx.cloud.Cloud({ resourceAppid, resourceEnv })` 访问资源方云环境，实现数据库和云函数的跨账号共享。

**初始化流程（app.js `_initResourceCloud`）：**
1. 创建跨账号 Cloud 实例并缓存
2. 通过跨账号实例调用 `login` 云函数验证资源方环境
3. 成功后设置 `_resourceCloudReady = true`

**数据访问方式：**
- `app.db()` — 获取资源方数据库实例
- `app.whenCloudReady()` — 等待云端初始化（含10秒超时保护）
- `app.callRepair(action, data)` — 调用聚合云函数，自动注入 shopPhone 和 clientOpenid

## 四、云函数

### 4.1 repair_main（聚合路由）

**调用方式：**
```js
wx.cloud.callFunction({
  name: 'repair_main',
  data: { action: 'createOrder', plate: '粤B12345', ... }
})
```

**统一返回格式：**
```json
{ "code": 0, "msg": "成功信息", "data": { ... } }
```
`code: 0` 成功，其他为失败。常见错误码：`-1` 参数缺失，`-2` 记录不存在，`-3` 无权操作，`-99` 系统异常。

**action 列表（15个）：**

| action | 功能 | 关键参数 | 安全校验 |
|--------|------|---------|---------|
| registerShop | 门店注册 | name, phone | 名称/手机号唯一性 |
| activatePro | Pro版激活 | code | unlockKey匹配+有效期 |
| addCar | 新增车辆 | plate, carType, ... | 车牌唯一性（按门店） |
| addMember | 新增会员 | plate, ownerName, phone, benefits | 已存在则追加权益 |
| createOrder | 创建工单 | plate, serviceItems, totalAmount, payMethod | 自动创建/更新车辆档案 |
| editOrder | 编辑工单 | orderId, updateData | 门店归属校验 |
| voidOrder | 作废工单 | orderId | 门店归属校验 |
| getDashboardStats | 看板统计 | shopPhone | 分页全量获取 |
| saveCheckSheet | 保存查车单 | plate, checkItems | — |
| updateCarInfo | 更新车辆 | docId, updateData | 门店归属+字段白名单 |
| updateMember | 更新会员 | docId, updateData | 门店归属 |
| useBenefit | 使用权益 | memberDocId, benefitIdx, newRemain, plate | 门店归属+扣减+生成工单 |
| updateOpenid | 绑定openid | docId | — |
| updateShopInfo | 更新门店 | field, value | 字段白名单（name/shopTel/shopAddr） |
| getOpenId | 获取openid | fetchId | 读取后删除临时记录 |

**关键实现细节：**
- `fetchAllOrders(where, field)` — 分页全量获取工单，突破单次100条限制
- `getOpenId` — 通过客户端 add 临时记录 → 云函数读取 `_openid` → 删除临时记录
- openid 优先使用 `event.clientOpenid`，解决跨账号场景获取问题
- `getDashboardStats` 到期提醒范围：-30天（已过期30天）~ +45天

### 4.2 login（独立云函数）

通过跨账号实例调用，用于验证资源方云环境连通性。

### 4.3 cloudbase_auth（资源方鉴权）

## 五、核心业务逻辑

### 5.1 工单状态流转

```
暂存工单 ──────────→ status: '施工中'（金额可为空）
保存工单（现付）──→ status: '已完成'（金额必填 > 0）
保存工单（挂账）──→ status: '待结算'（金额必填 > 0）
权益核销 ─────────→ status: '已完成'（金额为0，自动生成）
作废工单 ─────────→ isVoided: true（不删除记录，不参与统计）
```

### 5.2 Pro 版状态判断（统一规则）

```
code 有值 且 expireTime 未过期 → Pro 有效
code 有值 且 expireTime 已过期 → Pro 失效
code 无值 → 未激活
```

> 注意：激活后 `unlockKey` 被清空，不能用 `code === unlockKey` 判断激活状态。

### 5.3 自动登录流程

```
app.onLaunch
  → wx.cloud.init + _initResourceCloud
  → _checkPrivacyAgreement（未同意→隐私页）
  → _autoLogin → getOpenId()
  → _loadShopByOpenid（用openid查repair_activationCodes）
  → 有记录 → _restoreShopInfo（恢复缓存+判断Pro）→ 进入首页
  → 无记录 → _goWelcome('login') → 登录页
```

### 5.4 营收统计规则

- **参与统计**：`isVoided != true` 的所有工单（含"已完成""待结算"）
- **不参与统计**：`isVoided == true` 的工单
- 看板统计使用 `fetchAllOrders` 分页全量获取，确保超过1000条时统计准确
- 报表页同样分页全量获取

